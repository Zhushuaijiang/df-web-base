# 介绍

df-web-base 是面向 HIS（医院信息系统）场景的 Vue 3 组件库，基于 Vue 3、TypeScript 和 DevExtreme 构建，用于承载表单、表格、布局、反馈、导航以及部分业务组件能力，替代旧版 df-ui（Vue 2 + Element UI）。

## 项目定位

- 面向医疗信息系统的中后台组件库，覆盖通用 UI 与部分 HIS 业务场景。
- 同时提供 DevExtreme 封装组件、纯样式组件、复合布局组件和业务辅助能力。
- 支持全量注册、按需导入、UMD 构建以及微前端场景集成。

## 能力概览

- **组件与布局能力**：当前已提供较完整的组件、布局与数据展示能力，覆盖表单、表格、导航、反馈、图表、布局等常见页面场景。
- **DevExtreme 封装**：核心数据录入、数据展示和可视化能力基于 DevExtreme 封装，同时保留项目层需要的业务扩展接口。
- **TypeScript 优先**：组件、Hooks、工具函数与插件接口均以 TypeScript 类型为基础组织。
- **HIS 扩展接入**：提供字典、权限、租户、状态持久化等接入点，由宿主系统注入具体实现。
- **统一设计令牌**：基于 `--df-*` CSS 变量组织主题色、间距、圆角、字号等设计令牌。
- **文档与源码一致**：组件文档使用 Markdown 维护，可直接对应到源码与实际导出入口。

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue | 3.x | 响应式框架 |
| TypeScript | 5.x | 类型安全 |
| Vite | 5.x | 构建工具 |
| DevExtreme | 25.x | 核心 UI 引擎 |

具体依赖版本以 `package.json` 为准。

## 代码组成

- **components**：组件、布局组件和部分 HIS 业务组件的主要实现与导出入口。
- **hooks**：面向业务页面的组合式函数封装。
- **utils / constants / directives / event-bus**：通用工具、常量、指令和事件能力。
- **install / types**：插件注册入口与宿主系统接入协议。

## AI 使用指南

本文档所有页面均为 Markdown 源码，可直接被 AI 读取与解析。组件文档通常包含以下内容：

1. **组件描述** — 一句话说明用途
2. **基础用法** — 代码示例 + 效果说明
3. **Props 表格** — 属性名、类型、默认值、说明
4. **Events 表格** — 事件名、回调参数、说明
5. **Slots 表格** — 插槽名、作用域参数、说明
6. **注意事项** — 使用限制和最佳实践

AI 可优先读取 `docs/components/*.md` 获取组件 API 信息；全局接入方式与设计令牌可参考 `docs/guide/*.md`。
